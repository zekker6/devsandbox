Why devsandbox

Secrets stay yours

~/.ssh, ~/.aws, .env, parent directories, and other projects are invisible inside the sandbox. An AI agent can't exfiltrate what isn't mounted.

No more permission fatigue

Run with --dangerously-skip-permissions and actually mean it. Auto-approve commands without trusting your whole home directory.

Tool-agnostic

Claude Code, GitHub Copilot, aider, Cursor CLI, OpenCode — anything with a shell. Wraps the binary, no plugin needed.

How it works

  1. Wrap the command.

    Use devsandbox as a prefix. The directory you run it from becomes the sandbox root — everything else is blocked.

    cd ~/projects/my-app
devsandbox claude --dangerously-skip-permissions

  2. Filesystem locked down.

    Project files are read/write. Credentials, parent directories, sibling projects, and .env files are gone. .git is read-only by default.

    # inside the sandbox
ls ~/.ssh
# No such file or directory

cat .env
# (empty — masked with /dev/null)

  3. Network optional.

    Add --proxy to log every HTTP request. Block outbound entirely with a filter rule, or interactively approve each domain.

    devsandbox --proxy claude --dangerously-skip-permissions
devsandbox logs proxy --last 50
